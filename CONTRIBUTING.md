# Contributing to debug-EVM-tx-skills

## Pull Request Process

1. Fork the repository and create a branch from `main`.
2. Make focused changes to the skill package under `debug-EVM-tx-skills/`.
3. Keep documentation and examples consistent with the current skill name.
4. If behavior changes, update the relevant files in `debug-EVM-tx-skills/references/`.
5. Open a pull request with a concise summary of what changed and why.

## Contribution Guidelines

- One skill, one purpose.
- Keep the target audience non-technical.
- Prefer plain-language explanations over protocol jargon.
- Do not add secrets, API keys, or personal data.
- Do not introduce tools beyond Foundry, Heimdall, and Dedaub unless there is a strong reason.

## Good Contributions

- Better failure classification rules
- Clearer report formatting for non-technical users
- More accurate common error explanations
- Installation or documentation fixes
- Safer RPC and replay guidance

## Reporting Bugs

Use the bug report template and include:

- How you invoked the skill
- The model used
- The input and resulting output
- What you expected instead
